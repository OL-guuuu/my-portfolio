# Portfolio - Designer's Workspace

A cinematic editorial portfolio experience set in a designer's workspace.

---

## Project Overview

This is **not a traditional portfolio website**. It is a designed experience that immerses visitors in a UX/UI & Product Designer's creative workspace.

**Current Phase**: Phase 0 - Planning & Foundation

**Status**: Documentation and structure setup complete. No implementation yet.

## Key Principles

### World Coherence
- **One space only**: Designer's office/workspace
- **Monochrome**: Grayscale palette only
- **Premium**: High-end aesthetic
- **Restrained**: Intentional, not excessive
- **Editorial**: Magazine-quality composition
- **Cinematic**: Considered framing and storytelling

### Creative Reference
- **Lusion (https://lusion.co/)**: Reference for quality bar, atmosphere, restraint, cinematic pacing, and premium digital experience
- **NOT for copying**: Lusion is inspiration for quality and mood, not a template to clone
- **Our world remains**: Singular designer's office/workspace concept (original and unique)

### Workflow
- **Plan first**: No code before approval
- **One scope at a time**: Complete before expanding
- **Sequential quality**: No 3D before 2D is excellent
- **Preserve restraint**: Do less, make it better
- **Visible value**: User-perceivable quality only

### Quality Standards
- Semantic HTML
- WCAG 2.1 AA accessibility minimum
- Performance-conscious
- Responsive design
- No CSS frameworks (bespoke design)
- Progressive enhancement

## Contributing Guidelines

### Before Making Changes

1. **Read the documentation** (WORLD_BIBLE.md, WORKFLOW_RULES.md, CURRENT_PHASE.md)
2. **Check current phase** - Is your work phase-appropriate?
3. **Present a plan** - Get approval before implementing
4. **Stay in scope** - Do what's approved, nothing more

### Code Standards

- **HTML**: Semantic, accessible HTML5
- **CSS**: No frameworks, custom properties, mobile-first
- **JavaScript**: Vanilla JS preferred, progressive enhancement
- **Accessibility**: WCAG 2.1 AA minimum
- **Performance**: Fast loading, 60fps animations

### Commit Standards

- Clear, descriptive commit messages
- Reference phase/scope when relevant
- Conventional commits format encouraged
